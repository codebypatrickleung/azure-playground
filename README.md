# Azure Playground

This repository serves as my own sandbox for exploring and testing Azure services. It is intended for experimenting with new features, learning about Azure, and building applications that leverage Azure resources.

## What it does at the moment

This is my project to build a personal assistant that helps me stay on top of the latest news and developments, so I can quickly identify what I need to learn more about. This is powered by Azure OpenAI and LangGraph. Users search for a topic, pick categories (Cloud, Kubernetes, AI, etc.) and the app fetches the latest news, uses an LLM to filter for technical relevance, summarises each article in parallel, and lets you ask follow-up questions about any article.

## Infrastructure

All Azure infrastructure is provisioned with Terraform and can be torn down when no longer needed. The following resources are provisioned:

- **Azure Kubernetes Service (AKS)** — Standard load balancer, system-assigned managed identity
- **Azure Container Registry (ACR)** — Basic SKU, AcrPull role assigned to the AKS kubelet identity
- **Azure OpenAI** — model-router deployment (GlobalStandard, S0 SKU)
- **Azure Resource Group** — unique name generated by the `Azure/naming` module

## Application

### Frontend — Vue.js 3

- Topic search input with category filter chips (Cloud & Infrastructure, Kubernetes & Containers, AI & LLMs, Cybersecurity, Programming Languages)
- Animated step indicator while the LangGraph pipeline runs
- Article cards with source, date, LLM-generated summary, and relevance score (colour-coded)
- Deep dive panel — select any article and ask follow-up questions

### Backend — Python FastAPI

Three endpoints:

| Endpoint | Description |
|---|---|
| `POST /api/process-text` | Original general-purpose chat endpoint |
| `POST /api/news/search` | Runs the full LangGraph news pipeline |
| `POST /api/news/deep-dive` | Fetches article content and answers a follow-up question |

### LangGraph pipeline (`graph.py`)

| Node | What it does |
|---|---|
| `fetch_news` | Queries NewsAPI using topic + category keywords |
| `filter_articles` | LLM scores each article 1-10 for technical relevance, keeps ≥ 7 |
| `summarise_articles` | Summarises all filtered articles in parallel with `asyncio.gather` |

Authentication supports both managed identity (AKS) and API key (local dev) — detected automatically via the presence of `AZURE_OPENAI_API_KEY`.

## Helm & GitOps

The application is packaged as a Helm chart (`charts/azure-playground/`) and deployed to AKS via **Flux**. Git is the source of truth — pushing to `main` triggers reconciliation.

## CI / CD

Deployment is managed by `scripts/deploy.sh`, a modular bash script (macOS). Sections can be run individually or all at once.

| Section | What it does |
|---|---|
| `prereq` | Checks for required tools (`az`, `helm`, `flux`, `docker`, `kubectl`, `npm`) |
| `config` | Azure login, retrieves resource names |
| `build` | `npm ci`, Vite build, multi-arch Docker images, push to ACR |
| `deploy` | AKS credentials, namespace, RBAC for Azure OpenAI |
| `bootstrap` | One-time Flux bootstrap into AKS (skipped if already installed) |
| `run` | Creates the Helm values Secret, triggers `flux reconcile` |

## Security

- **Ephemeral infrastructure** — provisioned and destroyed as needed
- **Managed identities** — no hardcoded credentials; AKS uses a system-assigned identity to access Azure OpenAI
- **Secrets out of Git** — dynamic values (endpoints, API keys) stored in Kubernetes Secrets, not committed to the repository
- **CORS** — currently allows all origins (sandbox only, not production-ready)

## Prerequisites

- macOS with [Homebrew](https://brew.sh)
- Azure CLI (`az`)
- Terraform
- Docker Desktop
- A GitHub personal access token with `repo` scope (for Flux bootstrap)
- A [NewsAPI](https://newsapi.org) key (free tier is sufficient)

## Getting Started

### 1. Provision infrastructure

```zsh
cd infrastructure
terraform init
terraform apply -auto-approve
```

### 2. Deploy the application

```zsh
export GITHUB_TOKEN=<your-github-pat>    # repo scope required
export GITHUB_USERNAME=<your-username>

./scripts/deploy.sh --all
```

The script will:
- Check prerequisites and install any missing tools
- Authenticate with Azure and retrieve resource details
- Build and push Docker images to ACR
- Connect to AKS and configure RBAC
- Bootstrap Flux into the cluster (one-time)
- Create the Helm values Secret and trigger Flux reconciliation

The external IP is printed at the end. Open `http://<external-ip>` in your browser.

### 3. Run specific sections

```zsh
./scripts/deploy.sh --run <section>
# sections: prereq, config, build, deploy, bootstrap, run
```

## Local Development

Run the full stack locally without AKS using Docker Compose. Uses API key auth instead of managed identity.

```zsh
# Create a .env file
cat > .env <<EOF
AZURE_OPENAI_ENDPOINT=https://<your-endpoint>.openai.azure.com
AZURE_OPENAI_API_KEY=<your-api-key>
NEWS_API_KEY=<your-newsapi-key>
EOF

docker compose up --build
```

Open [http://localhost:8080](http://localhost:8080).

## Cleanup

```zsh
# 1. Remove the Helm release (cleans up LoadBalancer and public IP)
kubectl delete helmrelease azure-playground -n azure-playground

# 2. Uninstall Flux
flux uninstall --silent

# 3. Destroy Azure infrastructure
cd infrastructure && terraform destroy -auto-approve
```
